# PostPilot-AI Domain Context

## Core Concepts

### Draft
A container for a single content idea across multiple social platforms. It shares a `baseIdea` but allows platform-specific variants (LinkedIn, X) to have independent lifecycles.

- **Base Idea**: The core message or topic generated by AI.
- **Platform Variant**: The specific implementation of a Draft for a platform (e.g., a LinkedIn Post or an X Thread).
- **Independent Lifecycle**: Each Platform Variant has its own `status` and `scheduledAt` stored exclusively in dedicated top-level columns (e.g., `linkedinStatus`, `xScheduledAt`). This establishes a Single Source of Truth (SSOT), preventing data conflicts and allowing for high-performance indexing and atomic updates per platform.

## Glossary

| Term | Definition |
| :--- | :--- |
| **Draft** | The top-level entity (mapped to the `Post` table) containing all platform variants. |
| **Platform Variant** | A specific set of content and metadata (like `status` and `scheduledAt`) for LinkedIn or X. Can be partially deleted independently of the parent Draft. |
| **Relational SSOT** | The practice of storing lifecycle state and platform-specific content in dedicated table columns rather than a shared JSON blob. |
| **Post Pack** | A single Draft and its associated metadata generated by an AI prompt. |
| **X Thread** | A Platform Variant for X consisting of multiple sequential posts. |
| **Platform Status** | The lifecycle state (DRAFT, SCHEDULED, PUBLISHED, ARCHIVED) specific to a Platform Variant. |
| **Token** | A unit of daily credit. Each user receives 10 tokens per calendar day (UTC) for AI post generation. |
| **Token Transaction (Ledger)** | An append-only ledger entry recording token allotments, consumptions, and refunds per user per day. |
| **Token Allotment** | The daily +10 token grant, allocated lazily on the user's first token operation of the day. |
| **Token Consumption** | A -1 deduction from the user's balance, performed atomically before AI generation. |
| **Token Refund** | A +1 restoration to the user's balance when a generation fails, rate-limited at 10/min/user. |
